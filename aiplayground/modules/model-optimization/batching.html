Batching

Batching means processing multiple user requests together as one group, instead of one at a time. This can make model serving more efficient.

Simple Analogy

Cars vs a bus

Moving many people is much more efficient when they share one ride instead of each taking a separate car.

One car per person
Every person gets their own car. Lots of separate trips, lots of repeated effort.
= One request at a time
One bus for many
Many people share a single bus. One trip carries everyone — far more efficient.
= Batch processing

Important Trade-Off

Efficiency has a catch

Batching can improve efficiency, but waiting to form a batch can sometimes add a little delay for an individual request.

Batching improves throughput
but may affect latency
We explore this balance in the next lesson: Latency vs Throughput.

Key Takeaways

  • • Batching groups multiple requests together.
  • • It can make model serving more efficient.
  • • It is useful when many requests arrive.
  • • It can improve throughput.
  • • It may add waiting time for some requests.

Summary

Batching helps AI systems process multiple requests together. This can improve efficiency and throughput, especially when many users are using the model.
