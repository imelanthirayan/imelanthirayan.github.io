Normal vs Speculative

Two ways to reach the same words

Both produce quality text. They differ in who does the drafting and who does the checking — a bit like a junior writer and a senior editor.

Normal Decoding
  • The large model does all the work.
  • Generates one token at a time.
  • Reliable, but slower.
🖋️ = One careful editor writing every word
Speculative Decoding
  • A small model drafts several tokens.
  • The large model verifies them.
  • Faster when many drafts are accepted.
✍️ = Junior drafts fast, senior editor approves
Junior writer = small draft model  ·  Senior editor = large model  ·  Draft text = proposed tokens  ·  Accepted text = final output.
Why It Can Be Faster

Accept many at once

If the smaller model guesses several good tokens, the larger model can accept them together instead of generating each one slowly.

Accepted drafts = faster generation
Rejected drafts = correction needed

Key Takeaways

  • • Normal decoding generates tokens step by step.
  • • Speculative decoding uses a smaller draft model.
  • • The larger model verifies the draft tokens.
  • • Accepted draft tokens can speed up generation.
  • • Quality is protected because the larger model still checks the output.
